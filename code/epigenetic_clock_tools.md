# epigenetic_clock_tools.md

## 1. Цель файла

Здесь — практический обзор, **как использовать готовые эпигенетические часы** из публичных репозиториев, чтобы оценивать эпигенетический возраст и «пену» на мета‑уровне в реальных проектах. [github](https://github.com/mdozmorov/Aging_clock)

***

## 2. Основные репозитории

### 2.1. mdozmorov/Aging_clock

- Repo: `mdozmorov/Aging_clock`. [github](https://github.com/mdozmorov/Aging_clock/blob/master/README.md)
- Что есть:  
  - ссылки на основные статьи по эпигенетическим часам;  
  - примеры данных и скриптов;  
  - обзор разных моделей (Horvath, Hannum, PhenoAge, GrimAge и др.). [github](https://github.com/mdozmorov/Aging_clock)

Использование:

- хорошая отправная точка, чтобы понять, какие часы существуют и как они применяются.

### 2.2. liguowang/epical

- Repo: `liguowang/epical`. [github](https://github.com/liguowang/epical)
- «A repository of epigenetic age calculators»:  
  - сборник реализованных калькуляторов эпигенетического возраста;  
  - поддержка разных платформ данных (Illumina и т.п.).

Использование:

- если у вас есть сырые данные метилирования (матрица CpG × образцы), `epical` позволяет вычислить эпигенетический возраст по нескольким часам.

### 2.3. CastellaniLab/EpigeneticAgePipeline

- Repo: `CastellaniLab/EpigeneticAgePipeline`. [github](https://github.com/CastellaniLab/EpigeneticAgePipeline)
- Полный пайплайн:  
  - препроцессинг данных метилирования;  
  - расчёт эпигенетического возраста по ряду моделей;  
  - анализ эпигенетического ускорения.

Использование:

- для проектов, где нужно не только посчитать возраст, но и построить статистику по ускорению/замедлению старения.

### 2.4. jazoller96/mammalian-methyl-clocks

- Repo: `jazoller96/mammalian-methyl-clocks`. [github](https://github.com/jazoller96/mammalian-methyl-clocks)
- Коэффициенты и формулы для множества **млекопитающих** и других эпигенетических часов.

Использование:

- если вы работаете с животными (мыши, собаки и др.) и хотите оценивать эпигенетический возраст в экспериментах.

### 2.5. AkshajD/Epigenetic-Clock

- Repo: `AkshajD/Epigenetic-Clock`. [github](https://github.com/AkshajD/Epigenetic-Clock)
- Небольшой учебный проект:  
  - анализ метилирования на 6 CpG‑сайтах в крови;  
  - предсказание возраста с помощью ML.

Использование:

- чтобы понять на маленьком примере, как технически устроен расчёт часов.

## 3. Что нужно на вход

Чтобы использовать большинство часов, нужны:

- данные метилирования ДНК (обычно Illumina 27K/450K/850K чипы); [github](https://github.com/liguowang/epical)
- формат:  
  - beta‑значения CpG (0–1) или M‑значения;  
  - таблица с CpG по строкам и образцами по столбцам (или наоборот, в зависимости от пайплайна).

Минимум метаданных:

- возраст (для проверки/валидации);  
- пол, ткань, состояние (здоров/болен), интервенция (до/после) — для анализа.

## 4. Базовый workflow расчёта эпигенетического возраста

1. **Подготовка данных:**

   - нормализация и QC (часто уже реализованы в `EpigeneticAgePipeline` и других наборах); [github](https://github.com/CastellaniLab/EpigeneticAgePipeline)
   - фильтрация CpG по качеству и/или пересечение с набором CpG, используемых в выбранных часах.

2. **Выбор часов:**

   - для людей:  
     - Horvath, Hannum — если нужна хорошая привязка к хронологическому возрасту;  
     - PhenoAge, GrimAge — если интересен риск смерти и фенотипическое старение. [github](https://github.com/jazoller96/mammalian-methyl-clocks)
   - для животных:  
     - соответствующие часы из `mammalian-methyl-clocks`. [github](https://github.com/jazoller96/mammalian-methyl-clocks)

3. **Расчёт:**

   - использование готовых скриптов из `epical`, `EpigeneticAgePipeline` или других репо; [github](https://github.com/liguowang/epical)
   - получение на выход:  
     - эпигенетический возраст (years / species‑specific units);  
     - при необходимости — корректированный возраст (с учётом пола, ткани).

4. **Эпигенетическое ускорение:**

   - вычисление разницы:  
     - \( \text{Accel} = \text{Epigenetic Age} - \text{Chronological Age} \) для людей;  
     - или другие метрики (остаток регрессии, нормализация).

## 5. Как использовать в GRA-проектах

### 5.1. Оценка \(\Phi^{(M)}\) (пены мета-уровня)

- Эпигенетический возраст — числовой прокси **мета‑пены** \(\Phi^{(M)}\):  
  - выше возраст → больше потеря эпигенетической информации; [providence.elsevierpure](https://providence.elsevierpure.com/en/publications/loss-of-epigenetic-information-as-a-cause-of-mammalian-aging)
  - ускорение (Accel > 0) → система стареет быстрее хронологического времени.

### 5.2. Оценка эффекта интервенций

В проектах (на людях или животных):

- до/после интервенции (диета, нагрузка, фармакология, сенолитики, перепрограммирование): [pubmed.ncbi.nlm.nih](https://pubmed.ncbi.nlm.nih.gov/36638792/)
  - измеряем эпигенетический возраст;  
  - смотрим, уменьшилось ли ускорение, появился ли **эпигенетический откат**.

Это позволяет:

- количественно оценивать, насколько данное вмешательство уменьшает \(\Phi^{(M)}\);  
- сравнивать разные подходы (lifestyle vs фармакология vs комбинированные схемы).

## 6. Ограничения и аккуратность

Важно помнить: [github](https://github.com/mdozmorov/Aging_clock)

- разные часы ≠ одно и то же:  
  - одни лучше для календарного возраста, другие — для риска смерти;  
  - выбор модели должен соответствовать задаче.

- эпигенетический возраст — **биомаркер**, а не абсолютная истина:  
  - может меняться в зависимости от ткани, этнической группы, заболеваний;  
  - изменения в несколько лет могут быть в пределах ошибки/шума.

- для людей:  
  - использование часов для «оценки эффектов протокола» на одного человека требует осторожности;  
  - нужны повторные измерения и контекст (здоровье, лекарства, образ жизни).
