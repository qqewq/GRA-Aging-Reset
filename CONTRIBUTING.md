# CONTRIBUTING

Спасибо за интерес к проекту GRA-Aging-Reset.
Ниже — минимальные правила для контрибьюторов.

## 1. Формат вклада

Поддерживаются следующие типы вкладов:

- Теория: уточнения формализма GRA, новые разделы, альтернативные
  формулировки.
- Биология: правки и дополнения по старению, эпигенетике, сенесценции,
  системным сигналам с ссылками на peer-reviewed источники.
- Практика: улучшение описания стратегий образа жизни, карт препаратов,
  экспериментальных направлений (без превращения в "гайд по самолечению").
- Код: примеры пайплайнов для эпигенетических часов, анализ данных,
  симуляции GRA-модели.

## 2. Базовые принципы

1. **Без медицинских рекомендаций.**  
   Не добавляйте текст, который может интерпретироваться как прямое
   руководство к самолечению или назначению препаратов. Всё должно быть
   в формате "обзор / исследовательские направления".

2. **Опора на источники.**  
   Любое сильное утверждение по биологии, клинике или фармакологии
   желательно сопровождать ссылкой на публикацию (PubMed / PMC /
   крупные обзоры). Желательно BibTeX-запись в `references.bib`.

3. **Честность про статус.**  
   Для любых вмешательств (сенолитики, rapalogs, метформин,
   NAD+/sirtuins, перепрограммирование и т.п.) нужно явно указывать:
   - доклиника / ранняя клиника / большие РКИ / стандарт лечения,
   - известные риски / ограничения.

4. **Стиль.**  
   - Текстовые файлы — Markdown (`.md`) с простыми заголовками и
     структурой.
   - Без агрессивного маркетинга, хайпа и "кликабельных" формулировок.
   - Русский язык, при необходимости — дублирование ключевых терминов на
     английском.

## 3. Как отправить PR

1. Сделайте fork репозитория.
2. Создайте ветку от `main`, например `feature/your_change`.
3. Внесите изменения (теория/биология/практика/код).
4. При необходимости обновите:
   - `README.md` (если меняется общая структура),
   - `references.bib` (если добавляете новые статьи),
   - `pyproject.toml` / `requirements.txt` (если нужны новые зависимости).
5. Откройте Pull Request с кратким описанием:
   - что изменено,
   - какой уровень GRA затронут (теория, эпигенетика, сенесценция, системы, практика),
   - ссылки на ключевые источники, если есть.

## 4. Код и ноутбуки

- Располагайте код в `src/` или `notebooks/` (если будут добавлены).
- Старайтесь делать примеры:
  - воспроизводимыми (указать зависимости),
  - безопасными (без реальных медицинских рекомендаций и чувствительных данных).
- Не включайте в репозиторий приватные данные пациентов / людей
  (только открытые датасеты или анонимизированные данные с разрешением).

## 5. Этические ограничения

Не принимаются вклады, которые:

- продвигают DIY-использование онкопрепаратов, факторов Яманаки, генной
  терапии и других высокорисковых вмешательств;
- предлагают конкретные схемы приёма препаратов для "омоложения" без
  ссылок на серьёзные клинические исследования;
- содержат дискриминационный, оскорбительный или токсичный контент.

## 6. Обсуждения и идеи

Если у вас есть идея, но вы не уверены, как её правильно оформить:

- создайте Issue в GitHub с кратким описанием:
  - идея,
  - уровень (теория/биология/практика/код),
  - что вы хотите проверить или добавить.

Это поможет согласовать направление до того, как вы потратите много
времени на реализацию.

---

Контрибьютя, вы соглашаетесь, что ваш вклад может быть отредактирован,
упрощён или не принят, если он не соответствует целям проекта или
этим правилам.
